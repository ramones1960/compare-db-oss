DB 選択ウィザード

5つの質問に答えて、あなたのユースケースに最適なデータベースを見つけましょう。

Q1. データ構造は？
A 固定スキーマ・リレーション（ユーザー, 注文, 在庫など）
B 柔軟・ネスト・配列（スキーマが変わりやすい / JSON ライク）
C キーで直接取得（セッション・キャッシュ・ランキング）
D グラフ・つながり（ソーシャル・推薦・不正検知）
E 時系列データ（メトリクス・IoT・センサー）
F テキスト全文検索（商品検索・ログ解析・オートコンプリート）
G 大量集計・分析（BI・レポート・行動ログ集計）
H AI・意味検索（RAG・類似画像・推薦・ベクトル検索）
1 / 5
Q2. スケール要件は？
A 単一サーバで十分（〜数十万レコード・中小規模）
B 水平スケール（複数サーバ・大量データ・高トラフィック）が必要
2 / 5
Q3. 読み書き比率は？
A 読み取り中心（読み 7 : 書き 3 以上）— Web 閲覧・参照系
B 均等（読み 5 : 書き 5 前後）— 汎用 CRUD
C 書き込み中心（書き 7 : 読み 3 以上）— イベント・ログ収集
D 分析専用（書き込みバッチ + 集計読み取り）— BI・DWH
3 / 5
Q4. 整合性要件は？
A 強整合性 (ACID) — 金融・決済・在庫管理など
B 結果整合で問題ない — SNS・ログ・アクティビティフィード
4 / 5
Q5. クエリパターンは？
A 複雑な JOIN・集計・サブクエリ
B 単純な KV・プライマリキー検索（超高速・シンプル）
C 全文検索・スコアリング・ファセット
D 大量集計・BI・ダッシュボード・レポート
E 類似検索・ベクトル近傍・意味的マッチング
5 / 5
