📖 トランザクション分離レベルの解説

ACID の I（分離性 Isolation） は、並行するトランザクションが 互いの中間状態を見えないようにする度合いです。 分離レベルを高くするほど一貫性は高まりますが、スループットが下がります。

分離レベル Dirty Read Non-Repeatable Read Phantom Read PostgreSQL のデフォルト
Read Uncommitted 起きうる起きうる 起きうる—（PGはRCとして動く）
Read Committed 防止起きうる 起きうる✅ デフォルト
Repeatable Read 防止防止 起きうる*—
Serializable 防止防止 防止—（最も厳格）

* PostgreSQL の Repeatable Read は MVCC により Phantom Read も実質防止します。
各問題の体験デモは scenarios/isolation-levels/demo.py を参照してください。

Dirty Read: コミットされていない変更を別 TX が読む
Non-Repeatable Read: 同一 TX 内で同じ行を2回読むと値が変わる
Phantom Read: 同一 TX 内でのカウントが他 TX の INSERT で変わる 

-- 分離レベルの確認（PostgreSQL）
SHOW transaction_isolation;

-- TX 内で変更
BEGIN;
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;
SELECT count(*) FROM orders;   -- 100 件
-- 別セッションが INSERT + COMMIT
SELECT count(*) FROM orders;   -- Serializable: まだ 100 件（Phantom Read 防止）
COMMIT;
